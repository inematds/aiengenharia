MODULO 5.2

⚙️ Montando um mini-sistema

Do fluxo desenhado em papel ao fluxo executável num chat. Você aprende a encadear prompts (saída de um vira entrada do outro), criar loops de revisão e montar o sistema mínimo de 3 caixas — executável hoje, para qualquer profissão.

1

📄 Do fluxo no papel ao fluxo executável

O fluxo que você desenhou no módulo 5.1 já é um sistema. Agora vamos traduzir cada caixa em um prompt e executar a sequência numa sessão de chat. Sem instalar nada. Sem configurar nada.

🧩 Como funciona na prática

Na mesma sessão de chat: você envia o Prompt 1, recebe a saída, e usa essa saída como parte do Prompt 2 ("Com base no que você acabou de gerar, agora..."). A IA mantém o contexto da conversa — isso é o encadeamento.

Cada caixa do fluxo virou uma mensagem no chat. A sequência de mensagens é o sistema.

💡 Dica Prática

Para preservar o contexto entre caixas, comece cada prompt subsequente com: "Com base na [estrutura/análise/rascunho] anterior..." Isso ativa a janela de contexto da IA e garante que a saída do passo 1 seja a fundação do passo 2.

2

🔗 Encadeamento de prompts — exemplo completo de 4 etapas

A técnica mais importante do módulo. Cada etapa herda o trabalho da anterior e o transforma. Veja um exemplo completo de produção de artigo em 4 etapas encadeadas.

E1

Etapa 1 — Briefing e estrutura

Vou produzir um artigo sobre [TEMA] para [PÚBLICO]. Tom: [TOM]. Extensão: [X] palavras. Primeiro, gere um esboço com 5 seções e o objetivo de cada uma. Não escreva o artigo ainda.

Saída: esboço com títulos das seções + 1 frase descrevendo o conteúdo de cada uma. Isso é a entrada da Etapa 2.

E2

Etapa 2 — Primeiro rascunho

Com base no esboço que você gerou, escreva o artigo completo. Siga a estrutura das 5 seções. Tom: [TOM]. Não revise ainda — gere o rascunho completo.

Saída: artigo completo em rascunho. A instrução "não revise ainda" é intencional — revisão é a etapa seguinte.

E3

Etapa 3 — Avaliação contra critérios

Avalie o artigo que você gerou segundo estes 3 critérios: (1) Gancho nos primeiros 2 parágrafos? (2) Cada seção tem pelo menos 1 exemplo concreto? (3) Conclusão com CTA claro? Para cada critério: sim/não + o que precisa mudar.

Saída: diagnóstico estruturado. Se tudo "sim" → vai direto para a Etapa 4. Se há "não" → a Etapa 4 corrige apenas o que falhou.

E4

Etapa 4 — Versão final ajustada

Reescreva o artigo corrigindo os pontos que não passaram na avaliação. Mantenha tudo que já estava correto. Entregue a versão final completa.

Saída: artigo final, pronto para revisão humana mínima e publicação. Do briefing até aqui: 4 mensagens numa sessão.

Herança

Cada etapa parte do trabalho da anterior — não do zero

Separação de papéis

Geração e revisão em prompts separados = mais qualidade

Avaliação explícita

Critérios escritos antes — não julgamento após

Cirúrgico

Corrige só o que falhou — preserva o que estava bom

3

🗃️ Bases de conhecimento como peça do sistema

O sistema não começa do zero. Os ativos que você construiu nas trilhas anteriores são peças que se encaixam no fluxo. Injetá-los como primeira entrada é o que diferencia um sistema genérico do seu sistema.

COMO INJETAR A BASE DE CONHECIMENTO — ESTRUTURA DO PROMPT 0

# Contexto do sistema (injete sempre antes do Prompt 1)
Projeto: [nome do projeto — da T4]
Voz/tom: [do briefing-base T3]
Público: [ICP/perfil — da T3]
Templates aprovados: [da biblioteca T1]
Critérios de qualidade: [da rubrica T4]
Restrições: [da T3]

# Confirme que entendeu o contexto antes de eu começar o sistema.

Como a pirâmide se conecta aqui

  • T1 — Prompts prontos viram caixas do sistema
  • T3 — Briefing-base vira a entrada do sistema
  • T4 — Playbooks e rubricas viram critérios de avaliação
  • T5 (agora) — Conecta tudo num fluxo com loop
  • T6 (próximo) — O sistema aprende com cada ciclo

Por que injetar contexto no início

  • • Evita reexplicar quem você é e o que precisa a cada sessão
  • • Garante que cada caixa do sistema trabalhe com as mesmas premissas
  • • Transforma a IA de respondedora de perguntas em colaboradora do processo
4

🔄 Loop de revisão — a IA avalia o próprio resultado

Um loop de revisão é um prompt que pergunta à IA: "o que você acabou de gerar atende aos critérios X, Y, Z?" A resposta a essa pergunta direciona o sistema: se sim, entrega; se não, ajusta e repete.

TEMPLATE DE PROMPT DE AVALIAÇÃO (o "revisor" do sistema)

Avalie o [resultado] que você acabou de gerar segundo esta rubrica:
1. [Critério 1]: atende? sim / não → o que faltou
2. [Critério 2]: atende? sim / não → o que faltou
3. [Critério 3]: atende? sim / não → o que faltou

Se todos os critérios = sim:
Entregue o resultado final com uma linha: "✓ Todos os critérios atendidos."

Se algum critério = não:
Reescreva apenas o que não passou. Não altere o que já estava correto.

✓ Quando o loop funciona bem

  • ✓Critérios objetivos e verificáveis ("≤ 150 palavras", "tem CTA")
  • ✓Rubrica dada antes de o sistema rodar, não inventada após
  • ✓Máximo 3-4 critérios por loop (muitos critérios = paralisia)

✗ Quando o loop falha

  • ✗Critério vago: "seja criativo" → IA sempre dirá que atendeu
  • ✗Critério contraditório: o loop gira indefinidamente
  • ✗Sem critério de saída: o sistema nunca entrega
5

✅ Critérios de qualidade dentro do fluxo

Critério de qualidade não é o que você acha que é bom depois de ver o resultado. É o que você escreve antes de o sistema rodar. Essa distinção separa sistemas que convergem de sistemas que giram em loop infinito.

EXEMPLOS DE CRITÉRIOS OBJETIVOS VS. VAGOS

✗ VAGO (sistema não converge)

"Seja profissional"
"Tom adequado"
"Boa estrutura"
"Texto de qualidade"

✓ OBJETIVO (sistema converge)

"Tom direto, sem jargão corporativo"
"Máx. 150 palavras, 3 parágrafos"
"Primeira frase: pergunta ou dado"
"Termina com 1 ação concreta"

📊 Os 3 tipos de critério que todo sistema precisa

  • Critério de formato: tamanho, estrutura, número de seções, tipo de arquivo
  • Critério de conteúdo: elementos obrigatórios, o que não pode aparecer, referências necessárias
  • Critério de tom/voz: nível de formalidade, termos a usar/evitar, público-alvo do texto

💡 Dica Prática

Pegue um resultado que você aprovou no passado e pergunte: "por que aprovei isso?" As respostas são seus critérios. "Aprovei porque era curto, direto, tinha um exemplo e terminava com próximo passo" — isso são 4 critérios objetivos pra rubrica do seu sistema.

6

⚡ Atalho do retardatário — o sistema mínimo de 3 caixas

Antes de montar o sistema perfeito, monte o sistema mínimo. 3 caixas. Executável agora. No chat. É o MVP (mínimo produto viável) de processo — e ele já entrega valor real.

🧩 Esqueleto de fluxo — Kit T5

[CAIXA 1 — Entrada + contexto]
Injete o briefing-base. Dê o material de entrada. Defina o que quer como saída desta etapa.

[CAIXA 2 — IA gera proposta]
IA produz o resultado com base na entrada. Ainda não avalia.

[CAIXA 3 — Avaliação: critérios X + Y + Z]
→ ok: entrega o resultado final
→ não ok: ajusta cirurgicamente → volta à Caixa 2

📊 Por que 3 caixas são suficientes para começar

  • Você pode executar hoje. Não precisa de aprovação, configuração ou investimento.
  • O sistema mínimo revela o que melhorar. Depois de rodar 3 vezes, você sabe quais caixas precisam de mais detalhe.
  • A T6 (Evolução) se encarrega do resto. Você não precisa acertar o sistema na primeira versão — só precisa que ele exista.

Caixa 1 foca em

Contexto completo. Quanto mais rico, melhor a proposta da Caixa 2.

Caixa 2 foca em

Gerar sem avaliar. Separar geração de avaliação melhora os dois.

Caixa 3 foca em

Critérios objetivos. Essa caixa é o "controle de qualidade" do sistema.

7

👥 Na sua profissão — 5 fluxos prontos pra adaptar

O mini-sistema de 3 caixas funciona em qualquer área. Aqui estão 5 fluxos aplicados a 5 perfis — escolha o mais próximo do seu e adapte.

📚 Professor — fluxo de criação de aula

C1: Turma + currículo + objetivo da aula + conteúdo anterior
C2: IA gera plano de aula com objetivos, atividades e avaliação
C3: Avalia: alinhado ao currículo? ≤ 90 min? Tem exercício prático? → ok/ajusta

⚖️ Advogado — fluxo de análise de contrato

C1: Contrato anonimizado + contexto da operação + riscos que preocupam
C2: IA aponta cláusulas de risco, lacunas e sugestões de negociação
C3: Avalia: cobriu os 3 riscos prioritários? Linguagem técnica adequada? → ok/aprofunda

💼 Vendedor — fluxo lead → proposta

C1: Perfil do lead + histórico de contato + dor declarada + produto/serviço
C2: IA gera proposta personalizada com solução + benefícios + CTA
C3: Avalia: aborda a dor declarada? CTA claro? Tom correto pro perfil? → ok/ajusta

📊 Gestor — fluxo de decisão semanal

C1: KPIs da semana + problemas abertos + metas do trimestre + restrições
C2: IA sugere 3 prioridades para a semana com justificativa
C3: Avalia: alinhadas às metas? Dentro das restrições? Acionáveis? → ok/ajusta

✏️ Criador — fluxo ideia → post publicado

C1: Tema + público + tom da marca + 3 posts que performaram (referência)
C2: IA gera gancho + desenvolvimento + CTA no formato da rede
C3: Avalia: gancho para no scroll? CTA claro? Tom da marca? ≤ 150 palavras? → ok/ajusta

⚡ Atalho do retardatário — versão final

Escolha UM dos fluxos acima. Adapte a Caixa 1 com seu contexto real. Execute hoje numa sessão de chat. O primeiro sistema que funciona vale mais que o sistema perfeito que nunca saiu do papel. A T6 vai te ensinar a fazê-lo evoluir a cada ciclo.

⚙️ Resumo do Módulo

✓
Fluxo executável no chat — cada caixa é um prompt; a sequência de mensagens é o sistema
✓
Encadeamento de 4 etapas — briefing → estrutura → rascunho → revisão: saída de cada etapa alimenta a próxima
✓
Base de conhecimento injetada — T1 + T3 + T4 se tornam a entrada do sistema, não conhecimento ad hoc
✓

✓
Sistema mínimo de 3 caixas — Kit T5: Entrada → Gera → Avalia/Entrega. Executável hoje, sem ferramenta extra
✓
5 fluxos por profissão — professor, advogado, vendedor, gestor, criador: adapte o mais próximo do seu

📦 Entregável deste módulo:

Execute o mini-sistema de 3 caixas com um caso real do início ao fim. Documente: os 3 prompts que usou, os critérios de qualidade que definiu, o que passou e o que precisou de ajuste no loop. Esse documento é o seu mini-sistema documentado — o entregável da T5 e a base de entrada da T6.
